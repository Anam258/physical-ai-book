# Quickstart Guide: Physical AI & Humanoid Robotics Handbook - Part 1 Focus

This guide provides the necessary steps to initialize, develop, and deploy the Docusaurus-based Physical AI & Humanoid Robotics Handbook.

## 1. Initialize Docusaurus Project

To set up the Docusaurus v3 project in the current directory (repository root), run the following commands:

```bash
npm init docusaurus@latest . classic --typescript
npm install
```

This will create a new Docusaurus project with the classic template and TypeScript support.

## 2. Develop Chapters

Create your 14 chapters (Weeks 1-13 + Capstone) as Markdown or MDX files within the `docs/` directory. Follow these best practices:

*   **File Naming:** Use clear, descriptive filenames, optionally prefixed with numbers for explicit ordering (e.g., `01-introduction.md`, `13-capstone.mdx`).
*   **Front Matter:** Include `title` and `sidebar_position` in the YAML front matter of each Markdown/MDX file for proper display and ordering in the sidebar.
    ```markdown
    ---
    title: 'Week 1: Introduction to Physical AI'
    sidebar_position: 1
    ---

    # Week 1: Introduction to Physical AI

    Your chapter content here...
    ```
*   **MDX:** For interactive content or custom layouts, use MDX. Import and use React components directly within your `.mdx` files.
*   **Code Blocks:** Use fenced code blocks with language highlighting. Refer to the `research.md` for more details on advanced code block features (line numbering, highlighting).
    ```markdown
    ```typescript
    // Example TypeScript code
    const greeting: string = "Hello, Physical AI!";
    console.log(greeting);
    ```
    ```

## 3. Configure Docusaurus

Edit `docusaurus.config.ts` to configure the sidebar, hero section, footer, and GitHub Pages deployment. Refer to `research.md` for best practices regarding sidebar configuration.

### Sidebar Configuration (`sidebars.ts`)

Ensure your `sidebars.ts` (or `sidebars.js`) is set up to correctly display your 14 chapters. For autogenerated sidebars based on your `docs/` structure, it might look like this:

```typescript
const sidebars = {
  tutorialSidebar: [{
    type: 'autogenerated',
    dirName: '.',
  }],
};

export default sidebars;
```

### GitHub Pages Deployment

In `docusaurus.config.ts`, configure the `baseUrl` and `projectName` for GitHub Pages deployment. For a project hosted at `https://<USERNAME>.github.io/<REPO_NAME>/`:

```typescript
const config: Config = {
  // ... other configurations
  baseUrl: '/<REPO_NAME>/è ¶,
  projectName: '<REPO_NAME>',
  organizationName: '<USERNAME>',
  deploymentBranch: 'gh-pages', // or main, master, etc.
  // ...
};
```
Replace `<USERNAME>` and `<REPO_NAME>` with your actual GitHub username and repository name.

## 4. Build, Test, and Deploy

### Build

```bash
npm run build
```

This command will generate static content into the `build/` directory.

### Test Locally

```bash
npm run serve
```

This will serve your built site locally for testing. Verify the layout, navigation, mobile responsiveness, and content of all 14 chapters.

### Deploy to GitHub Pages

Once satisfied with the local build, deploy to GitHub Pages. Docusaurus provides a convenient script for this:

```bash
git add .
git commit -m "feat: Initialize Docusaurus handbook and content"
git push
GIT_USER=<YOUR_GIT_USERNAME> USE_SSH=true npm run deploy
```

Replace `<YOUR_GIT_USERNAME>` with your GitHub username. The `npm run deploy` command will build your site and push the `build` output to the `gh-pages` branch (or the branch you configured).
